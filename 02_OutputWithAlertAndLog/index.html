What's the difference between alert() and console.log()?

alert() creates a popup window with the supplied parameter that is readily visible to the user, while a console.log() prints out the parameter to the command line of the javascript console, which isn't readily apparent to the average user

What's one situation where you might use an alert() to tell the user something?

One common example I can think of is warning a user that a link they clicked will take them to a new website. Another example might be a popup explaining the rules of a game, or announcing a winner.

What's one way that alert() might get obnoxious?

Overuse seems like the most likely source of frustration from the user, as well as obstructing the main webpage

Would it make sense to use console.log() to tell the user things? Why or why not?

The Javascript console is the only way to see console.log() output, so it seems to be more of a developer tool than a method of communication to the user

What's one way that console.log() might be a good tool for you to use?

It seems like it will function as a debugging tool for doing things like checking variable values, conditional paths and other things that pry is used for in ruby
